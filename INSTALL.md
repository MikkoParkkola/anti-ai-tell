# Agent install runbook (install AND upgrade)

You are an AI agent installing or upgrading the anti-ai-tell writing
discipline for your user. The same steps cover both cases: every action below
overwrites the previous version's files, and the marker comments make stale
appended blocks replaceable. Re-running is always safe.

## 0. Upgrade note

If the user already has anti-ai-tell installed, this run upgrades it:
copied files are overwritten with the new version, and for APPEND targets
(AGENTS.md, copilot-instructions.md, .clinerules, GEMINI.md, CLAUDE.md,
.rules) you must first REMOVE the previous block, which is delimited by the start
marker `<!-- Generated by build_adapters.py` and the end marker
`<!-- /anti-ai-tell (end of generated block) -->`. Delete that whole span,
then append the fresh block. Never append a second copy.

## 1. Get the files

```sh
rm -rf /tmp/anti-ai-tell
git clone --depth 1 https://github.com/MikkoParkkola/anti-ai-tell /tmp/anti-ai-tell
```

## 2. Detect which clients the user has, install for each

Check for each directory or file below. For every one that exists, do the
matching install. Ask the user before writing into a project repo; user-level
locations need no confirmation.

| If this exists | Client | Install action |
|---|---|---|
| `~/.claude/` | Claude Code | `cp -r /tmp/anti-ai-tell/skills/anti-ai-tell ~/.claude/skills/` (or offer the plugin route: `/plugin marketplace add MikkoParkkola/anti-ai-tell` then `/plugin install anti-ai-tell@anti-ai-tell`) |
| `./.cursor/` | Cursor | `mkdir -p .cursor/rules && cp /tmp/anti-ai-tell/adapters/cursor-rules.mdc .cursor/rules/anti-ai-tell.mdc` |
| `./.github/` | GitHub Copilot | append `/tmp/anti-ai-tell/adapters/copilot-instructions.md` to `.github/copilot-instructions.md` (create if absent) |
| `./AGENTS.md` or user uses Codex CLI / Jules / Amp | AGENTS.md standard | append `/tmp/anti-ai-tell/adapters/AGENTS.md` to `AGENTS.md` (create if absent) |
| `./.windsurf/` | Windsurf | `mkdir -p .windsurf/rules && cp /tmp/anti-ai-tell/adapters/windsurf-rules.md .windsurf/rules/anti-ai-tell.md` |
| `./.clinerules` or user uses Cline/Roo | Cline / Roo Code | append `/tmp/anti-ai-tell/adapters/clinerules.md` to `.clinerules` |
| `~/.gemini/` | Gemini CLI | append `/tmp/anti-ai-tell/adapters/GEMINI.md` to `~/.gemini/GEMINI.md` |
| `./.aider.conf.yml` or user uses aider | Aider | `cp /tmp/anti-ai-tell/adapters/CONVENTIONS.md ./CONVENTIONS.md` and tell the user to add `read: CONVENTIONS.md` to `.aider.conf.yml` |
| `./.zed/` or user uses Zed | Zed | append `/tmp/anti-ai-tell/adapters/zed-rules.md` to `./.rules` |
| `./CLAUDE.md` (no plugin wanted) | CLAUDE.md projects | append `/tmp/anti-ai-tell/adapters/CLAUDE.md-snippet.md` to `CLAUDE.md` |

For ChatGPT (no filesystem): print the contents of
`/tmp/anti-ai-tell/adapters/chatgpt-custom-instructions.txt` and tell the user
to paste it into Settings → Personalization → Custom instructions.

For any client not listed: print `/tmp/anti-ai-tell/adapters/SYSTEM_PROMPT.md`
and tell the user to paste it into that client's system-prompt field.

## 3. Install the linter (optional but recommended)

```sh
mkdir -p ~/.local/share/anti-ai-tell
cp /tmp/anti-ai-tell/skills/anti-ai-tell/lint.py \
   /tmp/anti-ai-tell/skills/anti-ai-tell/vocabulary.json \
   ~/.local/share/anti-ai-tell/
```

Tell the user: `python3 ~/.local/share/anti-ai-tell/lint.py DRAFT.md` checks
any prose file; add `--prose` for emails and posts; `--json` for CI. Needs
Python 3.10+, no dependencies.

## 4. Verify

```sh
python3 /tmp/anti-ai-tell/skills/anti-ai-tell/lint.py /tmp/anti-ai-tell/tests/sample-ai.txt --prose
```

Expected: FAIL with about 12 findings (that file is deliberate AI-slop). Then:

```sh
python3 /tmp/anti-ai-tell/skills/anti-ai-tell/lint.py /tmp/anti-ai-tell/tests/sample-human.txt --prose
```

Expected: OK. If both behave as expected, the install works.

## 5. Report

Tell the user which clients you installed or upgraded, where the files went,
and that from now on their prose follows the three-tier discipline:
constraints at generation, `lint.py` before shipping, and the Tier-2 judgment
checklist in `SKILL.md` for high-stakes text. To upgrade later, the user
repeats the same one-line prompt; this runbook handles both cases.
